# Shadowverse Evolve Web Sandbox

このプロジェクトは、サーバーレス環境（PeerJSによるP2P通信）を利用して開発された、「Shadowverse Evolve」をウェブブラウザ上で遊ぶためのデジタルサンドボックス（盤面シミュレーター）です。
VPN接続やローカルネットワーク経由で、友人と完全無料でShadowverse Evolveを楽しむことを目的として設計されています。

---

## 🚀 アプリの起動と終了方法

### 必要な環境
- Node.js (推奨バージョン: v20 以降)
- npm (Node.jsに標準付属)
- Python 3.9 以降（カードデータの更新機能を使う場合のみ）

### Python 環境のセットアップ（初回のみ・カード更新機能を使う場合）
カードデータ更新スクリプト（`scrape_details.py` 等）を使用する場合は、以下のコマンドで Python の仮想環境を構築してください。

```bash
# プロジェクトのルートディレクトリで実行
python3 -m venv .venv

# 仮想環境を有効化（Mac / Linux）
source .venv/bin/activate

# 必要なパッケージをインストール
pip3 install -r requirements.txt
```

> **注意:** `.venv` フォルダは `.gitignore` で除外されています。`git clone` や `git pull` 後は毎回この手順で環境を再作成してください。

### 起動手順 (スタート)
1. ターミナル（Mac）やコマンドプロンプト（Windows）を開き、このスクリプトが入っているフォルダ（プロジェクトルート）に移動します。
2. 初回のみ、依存関係をインストールするために以下のコマンドを実行してください。
   ```bash
   npm install
   ```
3. ゲームを起動するには、以下のコマンドを実行します。
   ```bash
   npm run dev
   ```
4. ターミナル上に `➜  Local:   http://localhost:5173/` 等のURLが表示されるので、それをブラウザ（Chrome等）で開いてください。

### 終了手順 (ストップ)
1. ブラウザのタブを閉じてください。
2. サーバーを動かしているターミナルを開き、キーボードの **`Ctrl + C`**（Mac/Windows共通）を押します。
3. これでローカルサーバーが停止し、アプリが完全に終了します。

## 🎮 遊び方マニュアル

### 1. 部屋の作成と入室 (ルームマッチ)
このアプリはP2Pネットワークで同期されています。
- **ホスト側:** トップページの「Create Room」ボタンを押して部屋を作成します。割り当てられた数文字の英数字を「Room ID」として相手に伝えます。
- **ゲスト側:** トップページのテキストボックスに「Room ID」を入力し、「Join Room」を押して入室します。
※ 接続が成功すると、画面上部に「Guest connected! Game ready.」等の準備完了メッセージが表示されます。

### 2. デッキのインポート
このアプリは半オートのシミュレーターです。自分の手札・デッキは自分で管理します。
1. 画面右の「Deck Editor」から事前にデッキ（Main: 40~50枚, Evolve: 0~10枚）を作成し、**`.json` ファイルで保存** しておくことができます。
2. 実際のゲーム画面では、画面右下のコントロールパネルにある **「Import Deck (.json)」** をクリックし、保存した自分のデッキファイルを選択してください。
3. デッキが読み込まれ、自動的に「Main Deck」や「Evolve Deck」の領域にカードが裏向きでセットされます。

### 3. ゲームの基本操作（サンドボックス操作）
ルールエンジンは搭載されていませんので、実際の紙のカードゲームのようにマウスで自由にカードを動かして遊びます。

- **カードの移動:** カードをドラッグ＆ドロップして「My Field」「My EX Area」などの各ゾーンに移動させます。
- **ドロー:** 「Main Deck」をクリックして「Search」ボタンから選ぶか、右側のコントロールパネルの「Draw Card」を押すことで、山札からカードを手札（My Hand）に加えます（手札は相手からは裏向きになって見えません）。
- **マリガン・山札の下へ:** 引いたカードを右クリック（または左上をホバー）すると表示される「↓Bot」ボタンを押すと、カードがデッキの一番下へ戻ります。
- **カードの展開とスタック:** エボルヴ（進化）カードや、効果で直接重ねる必要があるカード（「にんじん」等）は、フィールド上の「ほかのカードの上」に直接ドラッグ＆ドロップすることで、**下に重なった状態（Stacked）**で表示されます。

### 4. 便利機能とショートカット操作
カードの操作を快適にするため、カードにカーソルを合わせると以下のショートカットボタンが表示されます。

- **`Tap` (タップ):** カードを90度傾けてレスト状態にします。もう一度押すとスタンドします。
- **`Flip` (裏返す):** カードを裏向きセット状態にします。
- **`EP+` `EP-` 等:** 「HP」や「攻撃力(ATK)」の左上カウンターを増減させます（バフ・デバフ用）。
- **`Token` (トークン生成):** 右側の「Spawn Token」をクリックすると、EXエリアにトークンカードがポンと生成されます。
- **`Banish` (消滅):** 場から直接「Banish（消滅）領域」へカードを送ります。公開状態で表示されます。
- **`To Evolve` (進化元へ戻す):** 場を離れた、使用済みの「エボルヴデッキ産のフォロワーカード」を**表向きで**対象のエボルヴデッキの上に返還します。（表向きの使用済み判定になります）

### 5. 相手のカードを奪う操作（所有権管理）
「簒奪の絶傑・オクトリス」や「カオティックエンジェル」など、相手のカードをEXエリアや場に持ってくる効果の場合は、**相手の盤面から直接相手のカードをドラッグして、自分の「My Field / My EX Area」に置く**ことが可能です。

また、相手のカードが場から離れる際に「Banish」や「↓Bot（山札の下）」を押したり、ドラッグでデッキへ戻そうとしたとしても、システム側で自動的に所有者を判定し、**「元の所有者（相手）のデッキや墓地」へ帰還**するよう制御されています。自分のデッキに混ざる心配はありません。

## 🔄 トークンの消滅ルールとゲームリセット

### トークンの完全消滅
紙の公式ルールに準拠し、トークン（Spawn Tokenボタンで発生させたカード等）は、フィールドおよびEXエリアにのみ存在できます。
トークンをドラッグ＆ドロップで「墓地・手札・消滅・デッキ」に入れようとした場合や、トークンに対して「↓Bot」「Banish」のショートカットを押した場合は、**システム上から完全にデータが消去**されます。

### リセット機能（Rematch・再戦）
同じルームで再戦したい場合、ページをリロードする必要はありません。
画面右下の赤い **「Reset Game」** ボタンを押してください。
専用の確認ダイアログ内で「Yes, Reset」を押すと、盤面や墓地、消滅エリアにある**すべてのカードが、自動的に元の持ち主の「Main Deck」や「Evolve Deck」の束に裏向きの状態で再編成**されます。
同時に、HPやPP、EP、SEP、ターンカウンターやコンボ数も初期設定に全てリセットされるため、そのままスムーズに次の試合を開始できます。

### 先攻・後攻の決定（コイントス）
画面上部のヘッダーバーにある **「🪙 Flip Coin (Decide Order)」** ボタンを押すことで、ランダムにあなたか対戦相手のどちらかが先攻に選ばれます。
先攻のEPは0、後攻のEPは3に自動設定されます。

## 🛠️ カードデータの更新手順（管理者・開発者向け）

公式のカードプールに新しいカードが追加された際、プロジェクトルートにあるPythonスクリプトを実行することで、アプリ内のカードデータベース（`public/cards_detailed.json`）を最新化できます。

**実行環境:** Python 3系 と `aiohttp`, `beautifulsoup4` ライブラリが必要です。

### 手順 1: 全カードの基本データ（名前・画像URL）を取得する
ターミナルを開き、プロジェクトのルートディレクトリで以下のコマンドを実行します。
```bash
python3 scraper.py
```
> これにより、公式サイトの検索一覧からすべてのカード情報が `public/cards.json` に上書き・追記保存されます。

### 手順 2: 各カードの詳細データ（コスト・クラス・テキスト等）を取得する
続いて、以下のコマンドを実行します。
```bash
python scrape_details.py
```
> このスクリプトは `cards.json` を読み込み、各カードの詳細ページを非同期通信で解析します。
> 取得したコストや収録弾のデータを含んだ最終的なデータベースが **`public/cards_detailed.json`** として保存されます。
> **注意:** APIへの負荷を避けるため、内部でバッチ処理とスリープを入れています。カード枚数によっては完了まで数分かかります。

### 手順 3: アプリへの反映
スクリプトの実行が完了したら、ブラウザのゲーム画面を**リロード**するだけです。Deck Builder画面に新しいカードが追加され、コスト検索等にも対応した状態で即座に使用可能になります。
